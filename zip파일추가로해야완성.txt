3.모든 생성자를 Uniform 초기화로 바꾸자 {}를 사용하는 방식으로 바꾸기

Uniform 초기화 => (균일 초기화) 

1) ()는 객체의 생성인지 함수의 선언인지 알 수 없음,
2) 또한 균일 초기화를 이용하면 축소 변환을 막을 수 있다. (explicit 생성자와 비슷하게 이해)
3) 그리고 생성자 호출이 복사 초기화가 아닌, 직접 초기화 방식으로 항상 처리된다.
4) 값 초기화 (기본 생성자 없는 멤버를 0 또는 기본 상태로 초기화)의 의미도 지닌다.

복사 호출 : MyClass My = myClass(...) // 임시객체생성 -> 복사 생성자
직접 호출 : MyClass My(...) //  직접 생성자
